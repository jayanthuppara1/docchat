# DocChat v1 - Scope Document

## 1. Problem

Users struggle to quickly understand key terms, risks, and obligations when reviewing long documents they need to sign, such as lease agreements, contracts, or terms and conditions.

## 2. User

The primary user for Version 1 is an everyday person who needs to review an important document before signing it, such as a lease agreement, service contract, policy document, or terms and conditions.

This user may not have the time, patience, or legal knowledge to read every page carefully, but still wants to understand what they are agreeing to, identify important clauses, and ask questions about confusing sections.

## 3. Core Value

The product helps users quickly understand important documents by letting them upload a PDF, get a summary, and ask questions before signing.

## 4. In Scope

### PDF Upload

Users should be able to upload a PDF document into the platform. The upload process should be simple and quick, allowing users to start working with their document immediately.

### Chat Interface

The platform should include a simple chat interface where users can interact with the uploaded PDF. Users should be able to type questions or requests related to the document.

### Instant Summary

Users should be able to generate a summary of the uploaded PDF. This helps them understand the main points of the document without reading the entire file manually.

### Document-Based Question and Answer

Users should be able to ask questions about the document and receive answers based on the PDF content. This allows users to quickly find key information, definitions, sections, or explanations from the document.

## 5. Out of Scope

The following features are not included in Version 1 because the focus is to build a simple and functional first version of the product.

### 1. PDF Editing

Users will not be able to directly edit the content of the uploaded PDF.

Features like changing text, modifying paragraphs, adding new sections, or editing the original document layout are not included in Version 1.

### 2. Document Viewer

Users will not be able to view the PDF inside the platform in Version 1. The focus is on the chat experience.

### 3. Exporting or Downloading Updated Files

Users will not be able to export, download, or save a modified version of the PDF from the platform.

Since document editing is not included in Version 1, exporting edited files is also out of scope.

### 4. Advanced Formatting Tools

Formatting features similar to Microsoft Word or Adobe PDF editor, such as font changes, text styling, page layout changes, tables, headers, footers, and design edits, are not included.

### 5. Annotations and Highlighting

Users will not be able to highlight text, draw on the PDF, add sticky notes, or leave comments inside the document viewer in Version 1.

### 6. Multiple File Uploads

Version 1 will support one PDF at a time.

Uploading and comparing multiple documents together will be considered for a future version.

### 7. User Accounts and Long-Term Document Storage

Version 1 will not include advanced user account management, saved document libraries, or long-term cloud storage.

The focus is on immediate upload and interaction with a document.

### 8. Collaboration Features

Real-time collaboration, shared document access, team comments, and multi-user chat are not included in Version 1.

### 9. Support for Other File Types

Version 1 will focus only on PDF files.

Other formats such as Word documents, PowerPoint files, Excel sheets, images, and scanned handwritten documents can be considered in future versions.

### 10. Advanced AI Features

Features such as automatic citation generation, document comparison, plagiarism checking, translation, advanced legal analysis, and custom report generation are not included in Version 1.

### 11. Mobile App

Version 1 will be built as a web-based experience.

A dedicated mobile application is not included in the current scope.
